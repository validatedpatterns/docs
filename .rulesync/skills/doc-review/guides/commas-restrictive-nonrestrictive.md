# Commas in Restrictive and Nonrestrictive Clauses

Format applicability: [All]
IBM Reference: Commas > Commas with nonrestrictive clauses and phrases
Severity: **Must-fix** for which/that misuse. **Recommended** for parentheses replacement.

## Rule

### Nonrestrictive clauses (extra info, removable)
- Use the relative pronoun **"which"**.
- **Surround with commas**.
- If you remove the clause, the basic meaning does not change.

### Restrictive clauses (essential, not removable)
- Use the relative pronoun **"that"**.
- **No commas**.
- The clause is essential to the sentence meaning.

## Examples

| DON'T | DO |
|---|---|
| The Recovery log which is generated automatically shows the cause. | The Recovery log, which is generated automatically, shows the cause. |
| The Recovery log, that contains the most recent information, is in the folder. | The Recovery log that contains the most recent information is in the folder. |

## Parentheses tip

Parenthetical text in running prose often signals a nonrestrictive clause. Replace parentheses with commas for better readability and translation.

| Avoid | Preferred |
|---|---|
| HTTP servers (such as Apache) often generate self-referential URLs. | HTTP servers, such as Apache, often generate self-referential URLs. |
